# 기여 가이드

## 1. 기여 원칙

이 프로젝트는 자동 검증 가능성과 재현성을 최우선으로 한다. 기능 추가보다 게이트 무결성과 문서 정합성을 먼저 확인한다.

## 2. 브랜치 전략

- 기본 브랜치: `main`
- 기능 브랜치 예시: `feature/<주제>`
- 버그 수정 브랜치 예시: `fix/<주제>`

## 3. 커밋 규칙

권장 접두사:

- `feat:`
- `fix:`
- `docs:`
- `test:`
- `refactor:`
- `release:`

커밋 메시지는 변경 이유를 포함해야 한다.

## 4. 코드 품질 원칙

1. 타입 힌트 유지
2. 예외 메시지 한국어 명시
3. 실패 상태 은닉 금지
4. 무차별 대입을 해결로 포장 금지

## 5. 문서 품질 원칙

1. 문서는 한국어로 작성
2. 금지 표현 사용 금지
3. 계산 결과의 한계를 분명히 표기
4. 형식화 실패 구간을 숨기지 않음

## 6. 테스트 원칙

- 단위 테스트, 통합 테스트, 회귀 테스트, 벤치마크 테스트를 유지
- 새 기능에는 최소 하나의 테스트를 추가

## 7. PR 체크리스트

- [ ] `pytest -q` 통과
- [ ] `ruff check` 통과
- [ ] 문서 업데이트 완료
- [ ] 번호 동기화 규칙 위반 없음
- [ ] 릴리즈 산출물 영향도 검토 완료
