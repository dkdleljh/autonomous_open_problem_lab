# 완전 무인 운영 정책

## 1. 기본 원칙

1. 수동 승인 단계 없이 상태 머신과 게이트로만 전이한다.
2. 실패 상태를 숨기지 않고 즉시 BLOCKED로 전이한다.
3. 모든 판정 근거를 감사 로그에 남긴다.

## 2. 사람 개입 금지 원칙

- 파이프라인 중간 단계에서 수동 확인 버튼을 두지 않는다.
- 승인 요청 메시지를 발생시키지 않는다.
- 사람이 개입할 수 있는 지점은 설정 수정과 재실행으로 제한한다.

## 3. 자동 중지 기준

다음 조건 중 하나라도 발생하면 해당 문제 실행을 즉시 중지한다.

- 중대한 논리 공백
- 강한형 반례 발견 후 약화형 권고 부재
- 참고문헌 충돌 미해결
- 형식화 미해결 의무 임계값 초과
- Paper QA 번호 불일치
- Release 아티팩트 누락

## 4. 자동 재시도 기준

- 재시도는 단계당 최대 2회
- 재시도 간 대기 시간은 `queue.yaml`의 `backoff_seconds` 값 사용
- 중대 실패는 재시도하지 않고 즉시 차단

## 5. 자원 예산 정책

`configs/global/runtime.yaml` 기준:

- 문제당 CPU 시간 예산
- 문제당 메모리 예산
- 문제당 최대 실행 시간

예산 초과 시 해당 문제를 격리 상태로 전환하고 기록만 남긴다.

## 6. 감사 추적 정책

- 모든 게이트 판정은 `pipeline_audit.jsonl`에 기록
- 검증 결과는 `verification_log.jsonl`에 별도 기록
- 마지막 실행 요약은 `last_run_summary.json`으로 저장

## 7. 안전 표현 정책

- 계산 결과는 실험적 지지로만 기록
- 형식화 실패 구간은 보고서에 명시
- 해결 주장 문구는 검증 게이트 통과 결과에만 허용
