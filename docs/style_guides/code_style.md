# 코드 스타일 가이드

## 1. 기본 규칙

- Python 3.11 기준 타입 힌트 사용
- 함수는 단일 책임 원칙 준수
- 모듈 간 의존성은 명시적 import 사용

## 2. 파일명 규칙

- 소문자 스네이크 케이스
- 모듈 책임이 파일명에 드러나야 함

## 3. 주석 규칙

- 필요한 경우에만 간결하게 작성
- 코드로 표현 가능한 내용은 주석으로 반복하지 않음
- 로그 메시지는 한국어로 명확하게 작성

## 4. 에러 처리 규칙

- 예외 삼키기 금지
- 실패 원인을 담은 명확한 예외 메시지 사용
- 자동 차단 조건에서는 즉시 실패 반환

## 5. 테스트 규칙

- 단위, 통합, 회귀, 벤치마크 테스트 분리
- 스키마 검증과 번호 동기화 검증 포함
