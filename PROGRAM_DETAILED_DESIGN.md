# 프로그램 상세 설계서

## 1. 설계 목표

이 시스템은 난제 자동 수집부터 제출 패키지 자동 생성까지를 단일 무인 파이프라인으로 통합한다. 설계의 핵심은 기능 추가보다 신뢰 가능한 전이 규칙과 감사 가능성이다.

## 2. 모듈별 책임

### 2.1 orchestrator

- 상태 머신 구동
- 품질 게이트 판정
- 단계별 이벤트 감사 로그 기록
- 실패 시 BLOCKED 전이

### 2.2 harvester

- 문제 후보 수집
- 수집 스냅샷 저장
- 중복 제거

### 2.3 registry

- canonical ID 생성
- alias 저장
- 상태 이력 관리

### 2.4 normalizer

- 객체, 가정, 목표 추출
- 동치형, 약화형, 강화형 생성
- normalized JSON 저장

### 2.5 scorer

- 5개 지표 계산
- 가중합 산출
- 선택 여부 판정

### 2.6 counterexample_engine

- 강한형 반례 탐색
- 탐색 범위, seed, 시간 기록
- 약화형 권고 생성

### 2.7 proof_engine

- 보조정리 후보 노드 생성
- 환원 경로 생성
- proof DAG 직렬화

### 2.8 verifier

- 금지 표현 탐지
- DAG 무결성 검사
- 문헌 충돌 검사
- 판정 보고서 생성

### 2.9 formalizer

- proof DAG를 Lean skeleton으로 변환
- import 자동 생성
- obligation 추적
- 빌드 시도 로그 기록

### 2.10 paper_generator

- 공통 의미 그래프 생성
- 한영 논문 동시 생성
- 번호 동기화 검사
- 재현성 부록 생성

### 2.11 submission_builder

- zip 패키지 생성
- 소스 번들 생성
- 체크섬 생성
- 릴리즈 노트 생성

## 3. 입력과 출력

### 입력

- 문제 원본 데이터 JSON
- YAML 설정 파일

### 출력

- 정규화 데이터
- 점수 데이터
- 반례 탐색 보고
- proof DAG
- 검증 보고
- 형식화 보고
- 논문 초안
- 제출 패키지

## 4. 의존성 설계

- Python: 오케스트레이션과 문서 생성
- Rust: 반례 탐색 가속 확장 지점
- Lean: 형식화 브리지
- NetworkX: proof DAG 구조 검증
- JSON Schema: 데이터 검증 기준

## 5. 상태 머신

상태 목록과 전이 규칙은 `aopl/core/state_machine.py`에 정의한다. 모든 전이는 게이트 결과를 필요로 하며 예외 경로는 없다.

## 6. 실패 규칙

- 게이트 실패 시 즉시 BLOCKED
- 실패 사유를 로그와 상태 이력에 동시 기록
- 다음 단계는 실행하지 않음

## 7. 확장 포인트

1. source_connectors 확장
2. 새로운 스코어 함수 추가
3. 다양한 반례 탐색 전략 플러그인 추가
4. Lean 이외 형식검증기 브리지 추가
5. 저널별 논문 템플릿 추가

## 8. 무인 운영 보장 장치

- 자동 중지 조건을 설정 파일로 관리
- 단계별 재시도 횟수 제한
- 아티팩트 누락 시 릴리즈 차단
- 감사 로그 기반 사후 추적

## 9. 릴리즈 설계

- Semantic Versioning 적용
- 자동 릴리즈 노트 생성
- 태그 기반 GitHub Release 생성
- 인증 누락 시 명확한 실패 처리
